Incompatibilities

  1. Changed: IBAMR no longer stores redundant component output: e.g., for INSStaggeredHierarchyIntegrator, the output velocity is stored by just U and not also U_x, U_y, and U_z. If you needed to access these values outside of VisIt or ParaView for data-processing purposes, then use, e.g., the path /processor.00000/level.00001/patch.00016/U.01 instead of /processor.00000/level.00001/patch.00016/U_x to retrieve component output. For more information see The new visualization module in the manual.
    (David Wells, 2026/02/06)

Major Changes

  1. New: Added RestartCleaner, a utility class that automatically manages restart directories to prevent disk space exhaustion in long-running simulations. It supports a "keep most recent N" cleanup strategy.
    (Meihan Chen, David Wells, 2026/02/20)

Minor Changes

  1. Fixed: autotools no-libMesh builds now skip libMesh-only `examples/complex_fluids/ex2` and `examples/complex_fluids/ex4` in `make examples`, avoiding missing libMesh header compile failures while preserving existing behavior when libMesh is enabled.
    (Boyce, 2026/02/27)
  2. Adopted C++17 clang-format baseline formatting for repository-wide style normalization prior to SAMRAI compatibility migration.
    (Boyce Griffith and Codex, 2026/02/27)
  3. Adopted angle-bracket include formatting across IBAMR and IBTK sources, headers, examples, and tests, while preserving quoted includes for tests.h and same-directory local headers.
    (Boyce Griffith and Codex, 2026/03/02)
  4. Fixed: Updated CFINSForcing to allow for mutiple stress tensors to be evolved. Updated the logging to prefix the object name to all logged messages.
    (Aaron Barrett, 2026/03/03)
